Regression testing

An additional directory has been added specifically to support regression testing. The directory (sfcb/regressionTests) contains two shell scripts to build regression test providers (build.sh) and to invoke the test shell scripts (test.sh).

Build.sh will build the test providers and programs using make and the providerRegister and classSchemas files. After build is executed start sfcBroker and then run test.sh.

Build.sh and test.sh serially visits all sub-directories of sfcb/(sfcb/regressionTests). One can add additional regression tests by adding new sub-directories.

DMTF schema data

The regression test environment requires that the DMTF schemas have been loaded. Loading the DMTF mof files is accomplished by issuing the following commands:

  1.   cd $HOME
  2.   cd sfcb
  3. sudo sh getSchema.sh  # Notice this script requires internet access and may take some time to complete

Building and Executing regression tests

Building regression tests goes as follows:

  1.   cd $HOME
  2. cd sfcb/regressionTests
  3. . ./setEnv
  4. sh build.sh
  5. sh broker.sh    

Start a new shell and do the following:

  1. cd $HOME
  2. cd sfcb/regressionTests
  3. . ./setEnv
  4. sh test.sh

Building and Executing selective regression tests

Building selective regression tests is done by adding a test directory as a parameter to the build.sh  and test.sh, as follows:

  1. cd $HOME
  2. cd sfcb/regressionTests
  3. . ./setEnv
  4. sh build.sh LifeCycleIndications
  5. sh broker.sh  

Start a new shell and do the following:

  1. cd $HOME
  2. cd sfcb/regressionTests
  3. . ./setEnv
  4. sh test.sh LifeCycleIndications

Adding additional regression tests

Create a new test sub-directory. You can call it whatever you want but not lib, bin or schema, let’s call it MyTests. MyTests must contain a makefile and two additional sub-directories: tests and schema.

The makefile is called like this: “make –c MyTests make install”. The assumption is that when make is completed all binaries will have been placed in sfcb/regressionTests/bin and that all libraries will have been placed in sfcb/regressionTests/lib.

Schema data

Every regression test must have schema data; it will be used by build.sh to build a classSchemas file.

Any *.mof file found in the ./MyTest/schema directory will be incorporated into the classSchemas file. There must be at least one .mof file.

Registration data

Every regression test must have registration data; it will be collected by build.sh and appended to the providerRegister file.

Any *.reg file found in the ./MyTest/schema directory will be used. There must be at least one .reg file.

Test scripts

Test scripts must be placed in sfcb/regressionTests/MyTest/tests. Any *.sh file found in this directory will be invoked by test.sh. Test scripts should return 0 when successful or 1 when not. Test scripts can/must do whatever is needed to do the test and should generate diagnostic messages when needed.

Supporting scripts

Two scripts are made available to make test environments easier.

wbemcat is a simple Perl script that interfaces to the nc command. It accepts a DMTF conforming CIM-XML operation request. This request is augmented with a minimal HTTP header and sent to a CIMOM. It will wait for a response and write the response unaltered to stdout.

catdiff is a shell script that works together with wbemcat. It is used to send a .xml file containing a DMTF conforming CIM-XML operation request to a CIMOM using wbemcat and to compare its response with a file containing the expected response. When no response file is found (designated by the .OK file type) the response is stored as the .OK file and is assumed to be valid for later reruns of this test. 

Both programs are can be found in the sfcb/test directory. The scripts are used in test/Indications are for testing indication support. 

Adding SBLIM  cmpi-base providers for regression testing

A script has been added to donload the cmpi-base providers from the SBLIM web site. It uses curl to download the following SBLIM packages:
  1. sblim-indication-helpers-xx
  2. sblin-cmpi-base-xx
  3. sblim-wbemcli-xx
  4. sblim-testsuite-xx
These pacckages are downloaded into regressionTests/sblim by using the getSblimBase.sh scriupt.
The regressionTests/sblim direcory contains additional scripts and data to automatically build the cmpi-base providers.
When executing sh test.sh sbli the SBLIM testsuite for the cmp base providers will be invoked.

Downloading and building the cmpi-base regression suite goes as follows:
  1. cd $HOME
  2.   cd sfcb/regressionTests
  3. . ./setEnv
  4. sh getSblimBase.sh   # Notice: this script requires internet access and may take some time to complete
  5. sh build.sh sblim    # Notice: this script may take some time to complete
  6. sh broker.sh  

Start a new shell and do the following to perform the cmpi-base regression tests:

  1. cd $HOME
  2. cd sfcb/regressionTests
  3. . ./setEnv
  4. sh test.sh sblim
